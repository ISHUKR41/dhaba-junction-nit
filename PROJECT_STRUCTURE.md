## ğŸ“ Key Directories Explained

### `client/` - React Frontend

- Pure React 18 + TypeScript application
- Built with Vite for fast development and optimized production builds
- Uses Tailwind CSS for styling
- Renders to `dist/public/index.html` during build

### `server/` - Express Backend

- Node.js/Express server that serves the frontend
- Handles API routes with `/api` prefix
- Integrates with Vite for development HMR
- In production on Vercel, becomes a serverless function

### `shared/` - Shared Code

- TypeScript/Zod schemas used by both frontend and backend
- Ensures type safety across the application

### `attached_assets/` - Static Files

- Images for the restaurant menu
- Menu data in JSON format
- All assets are embedded in the final build

### `dist/` - Build Output

- **`dist/index.js`** - Compiled backend server
- **`dist/public/`** - Compiled React frontend
- Generated by `npm run build` command
- This is what gets deployed to Vercel

### `api/` - Vercel Serverless

- NEW folder created for Vercel deployment
- Vercel routes requests here automatically
- Handler that delegates to Express app

### How Vercel Deploys

1. Runs `npm run build`
2. Outputs to `dist/public/` for frontend
3. Outputs to `dist/index.js` for API
4. Routes all requests through `api/index.ts`
5. Express handles routing and serves static files
